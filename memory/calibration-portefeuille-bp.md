Objectif (posé le 2026-07-05) : faire correspondre le TRI/VAN de TOUS les projets ModelFlow à leur Business Plan.

État de départ : moteur `engine.ts` gelé, calé sur 2 ancres (Baugé TRI 12,04 %, Sigoulès TRI 7,45 %) via bancs de test. Digoin = fiche BP prête (`docs/FICHE_DIGOIN.md`, 14 cibles), jamais validée. 23 projets importés `draft_bizdev` (stade BizDev, non calibrés). L'utilisateur confirme que **les 23 ont tous un BP** et va insérer inputs BP complets + cibles dans `input_projet_pipe.xlsx`.

Méthode d'orchestration décidée : **sérialiser ce qui touche le moteur, paralléliser la saisie.** Un seul `engine.ts` → un seul entonnoir de fixes moteur, ganté par la non-régression Baugé+Sigoulès. Découpler diagnostic (fan-out, ne patche pas) et correction (série, au centre). Ordre : T0 socle + harnais → pilotes Digoin (Fixed) + 1 Tracker → vidange des gaps génériques (G7 démantèlement, généralisation MES/`curveIndexAn1`, validation courbe Tracker jamais confrontée à un BP) → fan-out des 22 restants.

Non-régression obligatoire à chaque modif moteur : `npx vitest run app/lib/finance` + `npm run build`, ancres inchangées.

**CAUSE du décrochage dette identifiée (vérif adverse 2026-07-05).** Le 1er diagnostic (« sizing dette pré-IS vs post-IS », GAP #1) est RÉFUTÉ : IS-P90 = 0 pendant le tenor (le BP ne retranche pas l'IS du CFADS de sizing — C_Financing r80=r81, r93 Taxes P&L=0), et router sur `cfadsP90AfterTaxKeuro` cassait 5/35 tests sans caler les projets. VRAIE CAUSE : **OPEX discrétionnaires (« engagements ») par projet/année, dans `Inp_Opération` (ligne « Total Engagements » par bloc projet), intégrés au BP via C_P90 r196-204 → « Total operational expenses PV », que MF n'route PAS.** MF sous-compte l'OPEX P90 → CFADS trop haut → dette sur-dimensionnée (+2-6 %, croissant avec la marge/agri). Fermeture ~56-66 % de l'écart (Sigoulès repasse sous ±1 %). Résidu = renouvellements O&M à paliers + biais dégradation queue (G8b), PAS l'IS. FIX : input OPEX série an-par-an (lumpy, front-loadé, NON indexé) injecté dans opexP50 ET opexP90Keuro ; optionnel null-default → ancres inchangées. ⚠️ Leçon : le 1er agent avait produit un diagnostic plausible-mais-faux ; la vérif adverse (test empirique + effet sur ancres) l'a cassé — toujours vérifier avant de toucher le moteur.
